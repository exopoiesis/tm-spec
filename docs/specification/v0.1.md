# TM-Spec v0.1 — Draft

> **Third Matter Specification** — a single human-readable metalanguage for describing
> structures, defects, environments, reactions, DFT calculations, sanity gates, and
> provenance in the Third Matter project.
>
> Date: 2026-05-09. Status: [DRAFT]. Pilot artefact: see
> `examples/mack_vfe_neb.tm.yaml`.

---

## Design principles

1. **Each section = one mature domain standard** (CIF, Kröger-Vink,
   PHREEQC, Antimony, NOMAD Metainfo, AiiDA). Do not reinvent.
2. **Sanity gates are the original part of TM-Spec.** Compiled from
   the project lessons-learned (compiled into sanity gates G01..G24). Every PR with DFT results must pass all gates with `pass: true`.
3. **Units alongside numbers** (`0.043 eV`, not `0.043`).
4. **`paper_quotable: bool`** — explicit flag indicating that chemist+physicist opus review has been passed.
5. **`status: PASS | PRELIMINARY | FAIL | RETRACTED`** — research is messy.
6. **YAML** for human readability; **JSONL** for grep/jq/LLM-agent pipelines.
7. **Provenance as a DAG** with human-readable names (parent IDs, not UUIDs).

---

## Where each section comes from

| Section | Source | What we borrow |
|---|---|---|
| `structure` | AFLOW prototype + extxyz + CIF | Compact label + explicit Wyckoff positions |
| `defects` | Kröger-Vink + pymatgen-defects | Equation as code |
| `magnetic` | magCIF + Bilbao MAGNDATA | propagation vector + magmoms |
| `environment` | PHREEQC keyword blocks | speciation + kinetics in one place |
| `reactions` | Antimony + BNGL | text rules + rate laws |
| `calculation` | NOMAD Metainfo + QCSchema | code-agnostic level-of-theory |
| `workflow` | AiiDA + atomate2 | endpoints + provenance graph |
| `results` | Materials Project schema | typed values + uncertainty |
| `sanity` | **our** project lessons-learned | gates as tests in code |
| `provenance` | AiiDA + git | parents/hashes/cost |

## Header

```yaml
spec: tm-spec/0.1
kind: NEBCalculation     # Structure | Defects | Magnetic | Environment |
                         # Reaction | NEBCalculation | MDCalculation |
                         # USCalculation | MetaDynCalculation |
                         # MLIPBenchmark | SanityReport | Provenance
id: tm.<system>.<setup>.<method>.<date>
schema_url: https://exopoiesis.github.io/tm-spec/0.1.json   # [HYPOTHESIS] published
```

**Kind convention (Q-TMSPEC-1):** each methodological sampling family
gets its own kind. NEB ≠ US ≠ MetaDyn ≠ plain MD, because each
has unique structural blocks (endpoints / windows / hills / trajectory).

## Sections

### 1. `structure` — statics

Source: AFLOW prototype + extxyz + CIF.

```yaml
structure:
  formula: Fe16S16                          # supercell composition (post-defect)
  prototype: AB_tP4_129_a_c                 # AFLOW: mackinawite tetragonal
  space_group: { number: 129, symbol: P4/nmm }
  cell:
    a: 3.687  Å
    c: 5.029  Å
    gamma: 90 deg
  wyckoff:
    Fe@2a: [0.00, 0.00, 0.00000]
    S@2c:  [0.00, 0.50, 0.23420]
  supercell: [3, 3, 1]
  pbc: [true, true, true]
  ref: structures/mack_136at_pristine.extxyz
```

**Required fields:** `formula`, `cell`, `pbc`.
**Recommended:** `prototype`, `space_group`, `wyckoff`, `ref`.

### 2. `defects` — Kröger-Vink

Source: Kröger-Vink notation + pymatgen-defects.

```yaml
defects:
  reactions:
    - "Fe_Fe^× → V_Fe''  + Fe(removed)"     # iron vacancy
    - "nil    → H_i^•   + e'"               # interstitial proton + electron
  charge_balance: |
    V_Fe'' + 2·H_i^•  →  neutral
  population:
    V_Fe:  { count: 1, conc: 1/144 }
    H_int: { count: 2, conc: 2/144, site: tetrahedral_cavity }
```

**Notation (Kröger-Vink):**
- Superscript: `•` (positive), `'` (negative), `×` (neutral), relative to ideal.
- Subscript: site (`Fe_Fe`, `V_S`, `H_i`).

### 3. `magnetic`

Source: magCIF + Bilbao MAGNDATA.

```yaml
magnetic:
  state: AFM-G                              # FM | AFM-{A,C,G} | NM | PM-itinerant | ferri
  collinear: true
  propagation_vector: [0, 0, 0]
  magmoms_uB:
    Fe[top]:    +3.5
    Fe[bottom]: -3.5
  surrogate_warning: "PBE FM as PM/itinerant proxy at 363 K"
```

**`surrogate_warning`** — critical for this project: at 298–363 K
mackinawite is paramagnetic, but in DFT AFM-G is used as a surrogate.

### 4. `environment` — PHREEQC-style

Source: PHREEQC keyword blocks.

```yaml

  T: 363.15 K

    Na: 0.5
    Cl: 0.5
  database: phreeqc/llnl.dat
  saturation_targets:
    mackinawite: equilibrium

```

### 5. `reactions` — Antimony-flavoured

Source: Antimony + BNGL.

```yaml
reactions:
  - id: R1
    label: Fe-chemisorption
    rule:  "H_aq^+ + Fe_Fe^× → (H–Fe)_Fe^×"
    rate:  "k1 * [H+] * θ_Fe_free"
    params:
      k1: { value: 1.0e-4, units: M^-1·s^-1, source: dft_neb@per-project-dft-protocol }
    Ea_eV: 0.043
  - id: R2
    label: Grotthuss-shuttle
    rule:  "H2O···H+ ⇌ H+···H2O"
    rate:  "k2 * θ_H2O_chain"
    params:
      k2: { value: 1.2e+11, units: s^-1, source: aimd@a-prior-run }
    Ea_eV: 0.18
```

### 6. `calculation` — NOMAD-like, code-agnostic

Source: NOMAD Metainfo + QCSchema.

```yaml
calculation:
  method: DFT                                # DFT | AIMD | MLIP | classical_MD | tight_binding
  level:
    xc: PBE+D3(BJ)
    basis: { kind: plane_waves, cutoff_Ry: 80, rho_cutoff_Ry: 640 }   # NOMAD enum (D-02)
    pseudopotential: GBRV-1.5 USPP
    smearing: { kind: marzari-vanderbilt, width_Ry: 0.02 }              # lowercase enum (D-01)
    # Optional NOMAD/LibXC interop layer (D-03):
    xc_libxc: [GGA_X_PBE, GGA_C_PBE]
    vdw: D3-BJ
    spin: collinear                          # none | collinear | non-collinear
    hubbard:
      Fe-3d: { U_eV: 4.0, formulation: Dudarev }
  k_points: { mesh: [3, 3, 4], shift: [0, 0, 0] }
  convergence:
    scf_Ry:        1.0e-8
    fmax_eV_per_A: 0.05
  code:
    name: QuantumESPRESSO                    # | CP2K | ABACUS | GPAW | JDFTx | VASP
    version: 7.3.1
    image: infra-qe-gpu:cc80
    mpirun: "mpirun --bind-to none -np 1 pw.x"
    npool: 4
```

**jq idioms:**
```bash
# All paper_quotable Ea values
cat *.tm.jsonl | jq 'select(.kind=="NEBCalculation" and .status=="PASS") | .Ea_eV'

# Sanity reports with failures

# Cost summary
cat *.tm.jsonl | jq 'select(.kind=="Provenance") | .cost_usd' | awk '{s+=$1} END {print s}'
```

## Practical benefits

- **Single SI file for paper #1** = `paper/SI/mack_neb.tm.yaml`. The reviewer
  sees structure + level of theory + gates + provenance in one place.
- **NOMAD upload is trivial** — all fields are mappable.
- **Datasets in `results/dft_datasets/`** become JSONL —
  `cat *.tm.jsonl | jq` replaces our ad-hoc Python parsers.
- **Subagents** (chemist, physicist, devops) receive strictly
  typed input — fewer hallucinations.

## Open questions [QUESTION]

- **Q-TMSPEC-1** [CLOSED 2026-05-09]: Should the `kind` extension for US/PMF be separate from `MDCalculation` with `enhanced_sampling`?
  **Answer:** YES, a separate `kind: USCalculation`. Introduced via the pilot `examples/w2_us_pmf.tm.yaml`. Reasons:
    1. Endpoints (NEB) vs windows (US) — structurally different units of analysis.
    2. US has three specific sections (`cv_definition`, `sampling`, `pmf_analysis`) that do not fit into `workflow` or `results` of the existing schema.
    3. `MetaDynCalculation` and `MDCalculation` will be analogous separate kinds (v0.2).
  Applied convention: each *methodological sampling family* gets its own kind.

- **Q-TMSPEC-3**: Intersection with QCSchema — is it worth building a `tm-spec → qcschema` exporter for compatibility with the MolSSI stack?

### Additional sections introduced via the w2_us_pmf pilot

US-specific sections (present only in `kind: USCalculation`):
- `cv_definition` — collective variable + analytical form + units + implementation reference.
- `sampling` — windows, centers, springs, per-window MD config, totals.
- `pmf_analysis` — WHAM/MBAR + convergence + reliability range.

Analogous sections are needed for:
- `kind: MetaDynCalculation` — `cv_definition` + `metadyn_protocol` (hill height/width, bias factor) + `fes_analysis`.
- `kind: MDCalculation` — `ensemble` + `thermostat` + `trajectory_analysis`.
- `kind: MLIPBenchmark` — `backends[]` + `reference_dataset` + `metrics`.
